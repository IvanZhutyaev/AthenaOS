# ✅ ИСПРАВЛЕНО: Запуск P2P синхронизации

## Проблема
P2P узел создаётся, но не запускается автоматически при старте сервера.

## ✅ Решение (РЕАЛИЗОВАНО)

Добавлен метод `start_p2p_sync()` в `athena-core/src/system.rs` и метод `start_background()` в `athena-sync/src/p2p.rs`.

### Что было сделано:

1. **Добавлен метод `start_background()` в `P2PNode`** - запускает обработку событий Swarm в фоновой задаче
2. **Добавлен метод `start_p2p_sync()` в `AthenaSystem`** - инициализирует P2P узел и запускает его в фоне
3. **Интегрирован вызов в `athena-cli/src/main.rs`** - P2P синхронизация запускается автоматически при старте сервера (если включена в конфиге)

### Как это работает:

1. При старте сервера вызывается `system.start_p2p_sync()`
2. Метод проверяет, включена ли P2P синхронизация в конфиге
3. Если включена - запускает прослушивание на указанном порту
4. Запускает фоновую задачу для обработки P2P событий (подключения, отключения и т.д.)

### Использование:

P2P синхронизация теперь запускается автоматически при старте сервера, если в конфиге `enable_p2p = true`.

```bash
# Запуск с P2P синхронизацией (по умолчанию включена)
cargo run --bin athena -- start --port 8080
```
